工作流程

从上传包到查看报告，四步完成自动化测试

1
1

上传测试包

在 Web 界面拖拽上传 APK/RPK 包，系统自动识别包类型和版本信息

2
2

创建设备任务

选择测试包和目标设备，平台根据负载均衡策略自动分配到最优设备

3
3

调用 Testcase 执行

平台以子进程方式调用 testcase 框架，传递设备参数，实时捕获输出日志

4
4

生成并展示报告

测试完成后自动生成 HTML 报告，在 Web 界面查看详细结果、日志和截图
